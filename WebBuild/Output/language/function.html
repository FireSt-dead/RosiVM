Function

Well its a function... Differs from methods by being defined in module instead of class. I dislike C# static classes. Functions would be used for example in min, max etc. defined in module math.

Function definition:

<Function> ::=
    <Visibility> 'function' <Name> <TypeParameters> l-paren <Arguments> r-paren <ReturnType>
    <StatementBlock>

For example:

module math {
  function min(x: number) => number {
    return x >= 0 ? x : -x;
  }
}

Be very careful not to write procedural code using functions. Classes are still good for you.
