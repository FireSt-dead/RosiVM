Class

Classes encapsulate data with operations. Basic building block in Object Oriented Programming. A class can extend a single base class to inherit its members. A class may implement multiple interfaces.

Class syntax:

<Class> ::=
    <Visibility> 'class' <Name> <TypeParameters>
    <ClassExtends> <ClassImplements>
    l-brac <ClassMembers> r-brac
<ClassMembers> ::= <ClassMember> <ClassMembers> | <>
<ClassMember> ::= <ClassField> | <ClassConstructor> | <ClassMethod>

For example:

  1. module geom {
  2.     public class Point3D extends Point2D implements Clonable, Serializable {
  3.         var z: number;
  4.     }
  5. }

Class Visibility

  • "public": the class has no access restrictions.
  • "protected": the class is visible only from within the module it is defined in.
  • "private": forbidden to use for classes!
  • omitted: defaults to public.

Class Name

The class definition includes a non qualified name. A fully qualified class name would include the name of the module the class is defined in. A class is always defined within a module.

Class Extends

A class can have a base class. The type provided must be an actual type (if generic - the type parameters must be well defined) based on class. The class can not extend primitive types or interfaces.

<ClassExtends> ::= 'extends' <Type> | <>

Class Implements

A class can implement zero, one or multiple interfaces. Each type provided must be an actual type (if generic - the type parameters must be well defined) based on interface. The class can not implement primitive types or classes.

<ClassImplements> ::= 'implements' <Types> | <>
